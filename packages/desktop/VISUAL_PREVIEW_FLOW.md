## Performance Considerations

### Event Frequency
- **Action Preview**: Once per action (typically 10-100 actions per script)
- **Progress Update**: Once per action
- **Total Events**: ~2 events per action

### Latency
- **Event Emission**: < 1ms (JSON serialization)
- **IPC Transfer**: < 5ms (local process communication)
- **UI Update**: < 16ms (60 FPS animation)
- **Total Latency**: < 25ms (imperceptible to users)

### Memory Usage
- **Event Size**: ~200 bytes per event
- **State Size**: ~500 bytes (current action + metadata)
- **Animation**: Native driver (no JS thread blocking)
- **Total Impact**: Negligible (< 1MB for typical scripts)
